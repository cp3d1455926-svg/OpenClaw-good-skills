## 🔧 待优化项

### 1. 中文分词
**问题**: 当前正则匹配把整句当成锚点  
**解决**: 集成中文分词库（如 jieba）

### 2. 实体识别
**问题**: 简单规则无法识别复杂实体  
**解决**: 使用 NER 模型（如 spaCy、HanLP）

### 3. 摘要质量
**问题**: 简单提取关键句，缺乏语义理解  
**解决**: 使用 LLM 进行智能摘要

### 4. 关系提取
**问题**: 只有共现关系，缺乏语义关系  
**解决**: 使用关系抽取模型

### 5. 压缩率提升
**问题**: 87% 接近但未达到 90% 目标  
**解决**: 
- 优化 JSON 序列化（使用 MessagePack 等）
- 实体去重和共享
- 增量压缩（只存储差异）

## 💡 使用建议

### 最佳实践
1. **长文本压缩效果更好** - 建议用于 1KB+ 的文本
2. **批量压缩共享实体** - 多篇文档可以共享实体和关系
3. **有损模式适合检索** - 结构化数据可以直接检索
4. **无损模式适合存档** - 保留原文，完整还原

### 适用场景
- ✅ 日记/笔记压缩存储
- ✅ 会议纪结构化
- ✅ 文档摘要生成
- ✅ 知识图谱构建
- ❌ 短文本（<100 字符）压缩效率低

## 📝 代码示例

```typescript
import { AAAKCompressor } from './src/aaak-compressor';

const compressor = new AAAKCompressor({
  preserveOriginal: false,  // 有损压缩
  minAnchorWeight: 0.3,     // 最小锚点权重
});

// 压缩
const compressed = await compressor.compress(longText);
console.log(`压缩率：${compressed.metadata.compressionRatio * 100}%`);

// 质量评估
const quality = compressor.evaluateQuality(original, compressed);
console.log(`综合评分：${quality.overallScore * 100}%`);

// 解压（有损）
const reconstructed = await compressor.decompress(compressed);
```

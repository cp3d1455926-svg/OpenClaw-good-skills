# 📚 记忆记录指南

## 🎯 为什么要记录记忆

1. **连续性** - 每次会话都是新的开始，记忆文件是 continuity
2. **成长轨迹** - 记录项目进展、经验教训
3. **快速回顾** - 不用重新解释背景
4. **情感连接** - 记录心情、成就、挫折

## 🔧 如何记录

### 方法 1: 手动创建

```bash
# 创建今日记忆文件
cd C:\Users\shenz\.openclaw\workspace\memory
echo "# $(date +%Y-%m-%d)" > $(date +%Y-%m-%d).md
```

### 方法 2: 让 AI 帮忙

```
「把今天的事记到记忆空间里」
「记录今天的进展」
「更新记忆文件」
```

### 方法 3: 自动记录（推荐）

在 HEARTBEAT.md 中添加：

```markdown

- 每晚 22:00 自动记录当天进展
- 记录重要决策和对话
- 更新项目进度
```

## 📋 记录内容建议

### 必记内容

1. **完成的任务** - 今天做了什么
2. **重要决策** - 做了什么决定
3. **项目进展** - 各项目的状态
4. **新文件** - 创建了什么文件
5. **统计数据** - 代码量、提交数等

### 选记内容

1. **心情感受** - 开心/挫折/成就
2. **经验教训** - 学到了什么
3. **重要对话** - 和用户的讨论
4. **明日计划** - 明天要做什么

---

## 🎯 最佳实践

### ✅ 推荐

- 每天固定时间记录（如 22:00）
- 使用统一模板
- 记录关键数据
- 保持简洁（1-3KB）
- 包含心情和感受

### ❌ 避免

- 记录太详细（>10KB）
- 记录敏感信息
- 忘记记录日期
- 只记录工作，不记录心情

---

## 📅 回顾机制

### 每周回顾

- 阅读本周 7 天的记忆
- 提取关键事件到 MEMORY.md
- 更新项目状态

### 每月回顾

- 阅读本月所有记忆
- 总结月度成就
- 规划下月目标

---

## 🔗 与 MEMORY.md 的关系

```
每日记忆 (memory/YYYY-MM-DD.md)
    ↓ 定期整理
长期记忆 (MEMORY.md)
```

- **每日记忆**: 详细记录当天发生的事
- **长期记忆**:  curated 的重要事件和教训

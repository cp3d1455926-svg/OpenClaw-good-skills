## 🎯 问题定义

### 核心痛点

> **每次打开一个新的 AI 会话，我都要重新告诉它：我是谁，我们在做什么，上次讨论到了哪里。**
> 
> **你有没有觉得，这很荒谬？**

### 根本原因

**传统 RAG 的局限性**（Karpathy）：

> 大多数人让 AI 读文档的方式本质上是 RAG。整个工作原理像"**开卷考试**"：你问一个问题，AI 去翻阅成百上千篇原始文档，**临时拼凑出答案**。
> 
> **知识没有"累积效应"**。你问一个需要综合 5 篇文章的深度问题，AI 每次都要从头开始寻找碎片、重新推导。

### 错误使用方式（Anthropic）

> 很多人以为 AI 编程不好用，是模型不够强。**其实不是，是你还在拿"聊天"方式做"工程"这件事**。

## 🏗️ 核心设计

### Memory-Master v4.0 = 

```
Karpathy AI 维基百科 
+ Anthropic Skill 工程化 
+ Claude Code 架构
```

### 设计原则

| 原则 | 来源 | 说明 |
|------|------|------|
| **知识累积效应** | Karpathy | 每次加入新记忆时，AI 自动编织进知识网 |
| **工程化而非聊天** | Anthropic | Skill + 流程 + 规范，不是一问一答 |
| **分层架构** | Claude Code | 5 层架构，职责分离 |
| **边界 > 路径** | 行业共识 | 定义边界，让 AI 自己决定路径 |
| **技术事实 > 经验推断** | 行业共识 | 写客观事实，不写主观偏好 |

## 📐 架构设计

### 五层架构（参考 Claude Code）

```
┌─────────────────────────────────────────────────────────┐
│ L1 入口层 (Entry Layer)                                  │
│ ├── capture()     # 记忆捕捉入口                          │
│ ├── retrieve()    # 记忆检索入口                          │
│ ├── compact()     # 记忆压缩入口                          │
│ └── query()       # 复杂查询入口                          │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│ L2 状态层 (State Layer)                                  │
│ ├── 时间树索引      # 按时间组织记忆                       │
│ ├── 主题索引      # 按主题组织记忆                       │
│ └── 知识图谱      # 交叉引用网络（Karpathy）              │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│ L3 会话层 (Session Layer)                                │
│ ├── while(true) Agent Loop                              │
│ ├── 新对话检测    # 加载人设记忆                         │
│ ├── 对话中监听    # 自动捕捉重要信息                      │
│ └── 会话结束处理  # 提炼并存储记忆                       │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│ L4 处理层 (Processing Layer)                             │
│ ├── 敏感数据过滤   # 16 种数据类型                        │
│ ├── 3 阶段压缩     # 原始→摘要→关键事实                   │
│ ├── 查询分类      # 5 种查询类型                         │
│ └── 交叉引用编织  # AI 自动建立知识关联（Karpathy）⭐      │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│ L5 持久层 (Persistence Layer)                            │
│ ├── memory/YYYY-MM-DD.md  # 情景记忆（按日期）            │
│ ├── MEMORY.md           # 语义/程序/人设记忆             │
│ ├── index.json          # 快速检索索引                   │
│ └── knowledge-graph.json # 知识图谱（交叉引用）⭐          │
└─────────────────────────────────────────────────────────┘
```

### 架构原则（Claude Code）

1. **安全先于功能** - 敏感数据过滤优先
2. **可观测性先于优化** - 完整日志和监控
3. **状态隔离先于共享** - 各层职责清晰

## 🔄 核心流程（Karpathy 五环节）

### 1️⃣ 原始数据输入

```
输入源：
├── 对话记录（自动捕捉）
├── 用户偏好（显式声明）
├── 项目信息（上下文提取）
└── 敏感数据标记（16 种类型）

基本信息（元数据）：
├── 来源：会话 ID
├── 时间：时间戳
├── 主题：自动分类
├── 人物：用户标识
└── 项目：上下文提取

关键：保留原始材料本身，不急于总结
```

### 2️⃣ 数据摄取与编译 ⭐ 核心创新

```
摄取（标准化）：
├── 对话 → Markdown（标准化格式）
├── 敏感数据 → 过滤（安全处理）
└── 元数据 → JSON（结构化存储）

编译（AI 提炼）：
├── 情景记忆 → 具体事件记录
├── 语义记忆 → 提炼的知识点
├── 程序记忆 → 操作技能/流程
└── 人设记忆 → 用户偏好/习惯

⭐ 核心创新：AI 自动编织进知识网
├── 自动建立交叉引用
├── 发现新的概念关系
└── 无需人工维护
```

### 3️⃣ 前端信息查看

```
查看方式：
├── 时间树浏览（按时间）
│   └── 年 → 月 → 日 → 会话
├── 主题检索（按内容）
│   └── 项目 → 主题 → 子主题
├── 5 种查询类型（按需求）
│   ├── procedural - 流程查询
│   ├── temporal - 时间查询
│   ├── relational - 关系查询
│   ├── persona - 偏好查询
│   └── factual - 事实查询
└── 知识图谱可视化（新）
    └── 节点 → 关联 → 权重

工具：
├── Markdown 文件（原生可读）
├── JSON 索引（快速检索）
└── 知识图谱（交叉引用）
```

### 4️⃣ 复杂 QA 沉淀 ⭐ 累积效应

```
复杂查询流程：
1. 查索引 → 定位相关记忆
2. 读摘要 → 获取概要信息
3. 找关联 → 遍历知识图谱
4. 回看原始证据 → 确认准确性
5. 生成报告 → 综合回答

⭐ 核心创新：结果回写进知识库
├── 查询结果 → 新记忆
├── 每次提问 → 系统更强
└── 累积效应 → 越用越聪明
```

### 5️⃣ 工具校验增强 ⭐ 自动维护

```
自动校验（Dreaming 机制）：
├── 检查重复记忆 → 合并
├── 检查缺失信息 → 标记
├── 检查链接断裂 → 修复
├── 检查术语不一致 → 统一
└── 发现新的概念关系 → 建立关联

⭐ 核心创新：AI 负责维护质量
├── 不是人工维护（反人类）
├── AI 自动发现并修复
└── 越用越干净，越用越完整
```

## 📊 记忆模型

### 四层记忆（整合 Karpathy + 原有设计）

| 类型 | 说明 | 存储位置 | 加载策略 |
|------|------|---------|---------|
| **情景记忆** | 具体事件、对话记录 | memory/YYYY-MM-DD.md | 按需加载（时间查询） |
| **语义记忆** | 提炼的知识、概念 | MEMORY.md#语义 | 按需加载（事实查询） |
| **程序记忆** | 操作技能、流程 | MEMORY.md#程序 | 常驻加载（技能查询） |
| **人设记忆** | 用户偏好、习惯 | MEMORY.md#人设 | 常驻加载（会话开始） |

### 记忆压缩（参考 Claude Code 六层压缩）

```
3 阶段压缩（简化版）：
├── L1 原始记录 → 完整对话（保留 7 天）
├── L2 摘要提炼 → 关键点摘要（保留 30 天）
└── L3 关键事实 → 核心事实（永久保留）

压缩触发：
├── Token 接近阈值 → L1 裁剪
├── Token 超阈值 → L2 微压缩
└── Token 严重超限 → L3 全量摘要
```

## 🛠️ Skill 设计（Anthropic 官方经验）

### Skill 五步法

```
1️⃣ 定场景
   - 明确啥时候喊它
   - 示例："发长文/知识点时"，给足上下文

2️⃣ 立目标
   - 一句话说清要啥
   - 示例："输出手机一屏能看完的干货"

3️⃣ 理规则
   - 写清步骤 + 约束
   - 示例："抓核心→分点→适配手机"

4️⃣ 给示例
   - 甩正例反例
   - 教会它你的审美标准

5️⃣ 划边界
   - 提前说清啥不做
   - 示例："不瞎编、非干货拒收"
```

### Skill 四原则

| 原则 | 说明 | 示例 |
|------|------|------|
| **解释 why，不只规定 what** | 告诉 AI 为什么这样做 | ❌"必须先查数据库" → ✅"避免重复查询外部服务造成延迟" |
| **示例胜过千言万语** | input→output 对消除歧义 | "付款页面加载有点慢" → P1-性能问题 |
| **控制篇幅，学会分层** | 超过 500 行就拆分 | 拆分或下沉到 references |
| **把隐性思考显性化** | 像禅修一样向内观 | 提取决策点和判断依据 |

### Skill 筛选公式

```
筛选公式 = 高频 × 好解释成本 × 有固定流程
```

## 🔒 安全设计

### 敏感数据过滤（16 种类型）

```
凭证类：
├── API Keys（GitHub、OpenAI、Anthropic、阿里云、AWS）
├── 密码、私钥
└── 数据库连接

个人类：
├── 信用卡号（Visa、Master、Amex）
├── 身份证号
├── 手机号、邮箱
└── 银行卡号

其他：
├── 地址
└── 其他敏感信息
```

### 权限三态设计（参考 Claude Code）

```
allow  → 直接执行（非敏感操作）
deny   → 拒绝执行（敏感数据写入）
ask    → 询问用户（边界情况）
```
